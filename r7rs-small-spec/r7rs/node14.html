Naming conventions

By convention, indexfile"?" "tt" aux ? is the final character of the names of procedures that always return a boolean value. Such procedures are called indexfile"predicates" "rm" main predicates. Predicates are generally understood to be side-effect free, except that they may raise an exception when passed the wrong type of argument.

Similarly, indexfile"!" "tt" aux ! is the final character of the names of procedures that store values into previously allocated locations (see section 3.4). Such procedures are called indexfile"mutation procedures" "rm" main mutation procedures. The value returned by a mutation procedure is unspecified.

By convention, ``indexfile"->" "tt" aux ->'' appears within the names of procedures that take an object of one type and return an analogous object of another type. For example, `=13`list->vector takes a list and returns a vector whose elements are the same as those of the list.

A indexfile"command" "rm" main command is a procedure that does not return useful values to its continuation.

A indexfile"thunk" "rm" main thunk is a procedure that does not accept arguments.
