Multiple-value definitions

Another kind of definition is provided by `=13`define-values, which creates multiple definitions from a single expression returning multiple values. It is allowed wherever `=13`define is allowed.





`=13`=0.5em(indexfile"define-values" "tt" main define-valuesformals expression) syntax

It is an error if a variable appears more than once in the set of formals.

Semantics: Expression is evaluated, and the formals are bound to the return values in the same way that the formals in a `=13`lambda expression are matched to the arguments in a procedure call.
