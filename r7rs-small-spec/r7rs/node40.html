Delayed evaluation





`=13`=0.5em(indexfile"delay" "tt" main delayexpression) lazy library syntax

Semantics: The `=13`delay construct is used together with the procedure indexfile"force" "tt" aux force to implement indexfile"lazy evaluation" "rm" main lazy evaluation or indexfile"call by need" "rm" main call by need. (delay expression) returns an object called a indexfile"promise" "rm" main promise which at some point in the future can be asked (by the `=13`force procedure) to evaluate expression, and deliver the resulting value. The effect of expression returning multiple values is unspecified.






`=13`=0.5em(indexfile"delay-force" "tt" main delay-forceexpression) lazy library syntax

Semantics: The expression `=13`(delay-force expression) is conceptually similar to `=13`(delay (force expression)), with the difference that forcing the result of `=13`delay-force will in effect result in a tail call to `=13`(force expression), while forcing the result of `=13`(delay (force expression)) might not. Thus iterative lazy algorithms that might result in a long series of chains of `=13`delay and `=13`force can be rewritten using `=13`delay-force to prevent consuming unbounded space during evaluation.






`=13`=0.5em(indexfile"force" "tt" main forcepromise) lazy library procedure

The `=13`force procedure forces the value of a promise created by indexfile"delay" "tt" aux delay, indexfile"delay-force" "tt" aux delay-force, or indexfile"make-promise" "tt" aux make-promise.indexfile"promise" "rm" aux If no value has been computed for the promise, then a value is computed and returned. The value of the promise must be cached (or ``memoized'') so that if it is forced a second time, the previously computed value is returned. Consequently, a delayed expression is evaluated using the parameter values and exception handler of the call to `=13`force which first requested its value. If promise is not a promise, it may be returned unchanged.

The following example is a mechanical transformation of a lazy stream-filtering algorithm into Scheme. Each call to a constructor is wrapped in `=13`delay, and each argument passed to a deconstructor is wrapped in `=13`force. The use of `=13`(delay-force ...) instead of `=13`(delay (force ...)) around the body of the procedure ensures that an ever-growing sequence of pending promises does not exhaust available storage, because `=13`force will in effect force such sequences iteratively.

          gobblecr(define (stream-filter p? s)  (delay-force   (if (null? (force s))        (delay '())       (let ((h (car (force s)))             (t (cdr (force s))))         (if (p? h)             (delay (cons h (stream-filter p? t)))             (stream-filter p? t))))))


(head (tail (tail (stream-filter odd? integers))))                               5

The following examples are not intended to illustrate good programming style, as `=13`delay, `=13`force, and `=13`delay-force are mainly intended for programs written in the functional style. However, they do illustrate the property that only one value is computed for a promise, no matter how many times it is forced.


unspecified error makeother^`=13` 


          gobblecr(define count 0)(define p  (delay (begin (set! count (+ count 1))                (if (> count x)                    count                    (force p)))))(define x 5)p                     a promise(force p)             6p                     a promise, still(begin (set! x 10)       (force p))     6

Various extensions to this semantics of `=13`delay, `=13`force and `=13`delay-force are supported in some implementations:

  • Calling `=13`force on an object that is not a promise may simply return the object.

  • It may be the case that there is no means by which a promise can be operationally distinguished from its forced value. That is, expressions like the following may evaluate to either #t or to #f, depending on the implementation:


    unspecified error makeother^`=13` 

    
          gobblecr(eqv? (delay 1) 1)          (pair? (delay (cons 1 2)))

  • Implementations may implement ``implicit forcing,'' where the value of a promise is forced by procedures that operate only on arguments of a certain type, like `=13`cdr and `=13`*. However, procedures that operate uniformly on their arguments, like `=13`list, must not force them.


    unspecified error makeother^`=13` 

    
          gobblecr(+ (delay (* 3 7)) 13)  (car  (list (delay (* 3 7)) 13))    a promise





`=13`=0.5em(indexfile"promise?" "tt" main promise?obj) lazy library procedure

The `=13`promise? procedure returns #t if its argument is a promise, and #f otherwise. Note that promises are not necessarily disjoint from other Scheme types such as procedures.






`=13`=0.5em(indexfile"make-promise" "tt" main make-promiseobj) lazy library procedure

The `=13`make-promise procedure returns a promise which, when forced, will return obj. It is similar to `=13`delay, but does not delay its argument: it is a procedure rather than syntax. If obj is already a promise, it is returned.
