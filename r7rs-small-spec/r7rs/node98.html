Standard Feature Identifiers

An implementation may provide any or all of the feature identifiers listed below for use by `=13`cond-expand and `=13`features, but must not provide a feature identifier if it does not provide the corresponding feature.

`=13`r7rs

All RRS Scheme implementations have this feature. `=13`exact-closed

All algebraic operations except `=13`/ produce exact values given exact inputs. `=13`exact-complex

Exact complex numbers are provided. `=13`ieee-float

Inexact numbers are IEEE 754 binary floating point values. `=13`full-unicode

All Unicode characters present in Unicode version 6.0 are supported as Scheme characters. `=13`ratios

`=13`/ with exact arguments produces an exact result when the divisor is nonzero. `=13`posix

This implementation is running on a POSIX system. `=13`windows

This implementation is running on Windows. `=13`unix, darwin, gnu-linux, bsd, freebsd, solaris, ...

Operating system flags (perhaps more than one). `=13`i386, x86-64, ppc, sparc, jvm, clr, llvm, ...

CPU architecture flags. `=13`ilp32, lp64, ilp64, ...

C memory model flags. `=13`big-endian, little-endian

Byte order flags. `=13`name

The name of this implementation. `=13`name-version

The name and version of this implementation.
