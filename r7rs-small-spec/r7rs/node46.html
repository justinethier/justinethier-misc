Binding constructs for syntactic keywords

The `=13`let-syntax and `=13`letrec-syntax binding constructs are analogous to `=13`let and `=13`letrec, but they bind syntactic keywords to macro transformers instead of binding variables to locations that contain values. Syntactic keywords can also be bound globally or locally with `=13`define-syntax; see section 5.4.





`=13`=0.5em(indexfile"let-syntax" "tt" main let-syntaxbindings body) syntax

Syntax: Bindings has the form
unspecified error makeother^`=13` 


          gobblecr((keyword transformer spec) )
Each keyword is an identifier, each transformer spec is an instance of `=13`syntax-rules, and body is a sequence of one or more definitions followed by one or more expressions. It is an error for a keyword to appear more than once in the list of keywords being bound.

Semantics: The body is expanded in the syntactic environment obtained by extending the syntactic environment of the `=13`let-syntax expression with macros whose keywords are the keywords, bound to the specified transformers. Each binding of a keyword has body as its region.

`=13`=0.5em(indexfile"letrec-syntax" "tt" main letrec-syntaxbindings body) syntax

Syntax: Same as for `=13`let-syntax.

Semantics: The body is expanded in the syntactic environment obtained by extending the syntactic environment of the `=13`letrec-syntax expression with macros whose keywords are the keywords, bound to the specified transformers. Each binding of a keyword has the transformer specs as well as the body within its region, so the transformers can transcribe expressions into uses of the macros introduced by the `=13`letrec-syntax expression.
