Exceptions

This section describes Scheme's exception-handling and exception-raising procedures. For the concept of Scheme exceptions, see section 1.3.2. See also 4.2.7 for the `=13`guard syntax.

indexfile"Exception handler" "rm" main Exception handlers are one-argument procedures that determine the action the program takes when an exceptional situation is signaled. The system implicitly maintains a current exception handler in the dynamic environment.

indexfile"current exception handler" "rm" aux The program raises an exception by invoking the current exception handler, passing it an object encapsulating information about the exception. Any procedure accepting one argument can serve as an exception handler and any object can be used to represent an exception.





`=13`=0.5em(indexfile"with-exception-handler" "tt" main with-exception-handlerhandler thunk) procedure

It is an error if handler does not accept one argument. It is also an error if thunk does not accept zero arguments.

The `=13`with-exception-handler procedure returns the results of invoking thunk. Handler is installed as the current exception handler in the dynamic environment used for the invocation of thunk.

          gobblecr(call-with-current-continuation (lambda (k)  (with-exception-handler   (lambda (x)    (display "condition: ")    (write x)    (newline)    (k 'exception))   (lambda ()    (+ 1 (raise 'an-error))))))        exception 		and prints  condition: an-error


(with-exception-handler (lambda (x)  (display "something went wrong\n")) (lambda ()  (+ 1 (raise 'an-error)))) 		prints  something went wrong

After printing, the second example then raises another exception.





`=13`=0.5em(indexfile"raise" "tt" main raiseobj) procedure

Raises an exception by invoking the current exception handler on obj. The handler is called with the same dynamic environment as that of the call to `=13`raise, except that the current exception handler is the one that was in place when the handler being called was installed. If the handler returns, a secondary exception is raised in the same dynamic environment as the handler. The relationship between obj and the object raised by the secondary exception is unspecified.





`=13`=0.5em(indexfile"raise-continuable" "tt" main raise-continuableobj) procedure

Raises an exception by invoking the current exception handler on obj. The handler is called with the same dynamic environment as the call to `=13`raise-continuable, except that: (1) the current exception handler is the one that was in place when the handler being called was installed, and (2) if the handler being called returns, then it will again become the current exception handler. If the handler returns, the values it returns become the values returned by the call to `=13`raise-continuable.


unspecified error makeother^`=13` 


          gobblecr(with-exception-handler  (lambda (con)    (cond      ((string? con)       (display con))      (else       (display "a warning has been issued")))    42)  (lambda ()    (+ (raise-continuable "should be a number")       23)))   prints: should be a number   65





`=13`=0.5em(indexfile"error" "tt" main errormessage obj ) procedure

Message should be a string.

Raises an exception as if by calling `=13`raise on a newly allocated implementation-defined object which encapsulates the information provided by message, as well as any objs, known as the indexfile"irritants" "rm" main irritants. The procedure `=13`error-object? must return #t on such objects.


unspecified error makeother^`=13` 


          gobblecr(define (null-list? l)  (cond ((pair? l) #f)        ((null? l) #t)        (else          (error            "null-list?: argument out of domain"            l))))





`=13`=0.5em(indexfile"error-object?" "tt" main error-object?obj) procedure

Returns #t if obj is an object created by `=13`error or one of an implementation-defined set of objects. Otherwise, it returns #f. The objects used to signal errors, including those which satisfy the predicates `=13`file-error? and `=13`read-error?, may or may not satisfy `=13`error-object?.






`=13`=0.5em(indexfile"error-object-message" "tt" main error-object-messageerror-object) procedure

Returns the message encapsulated by error-object.






`=13`=0.5em(indexfile"error-object-irritants" "tt" main error-object-irritantserror-object) procedure

Returns a list of the irritants encapsulated by error-object.

`=13`=0.5em(indexfile"file-error?" "tt" main file-error?obj) procedure

Error type predicates. Returns #t if obj is an object raised by the `=13`read procedure or by the inability to open an input or output port on a file, respectively. Otherwise, it returns #f.
