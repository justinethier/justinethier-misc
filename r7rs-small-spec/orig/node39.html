Iteration

`=13`=0.5em(do ((variable init step) syntax
indexfile"do" "tt" main )
(test expression )
command )

Syntax: All of init, step, test, and command are expressions.

Semantics: A `=13`do expression is an iteration construct. It specifies a set of variables to be bound, how they are to be initialized at the start, and how they are to be updated on each iteration. When a termination condition is met, the loop exits after evaluating the expressions.

A `=13`do expression is evaluated as follows: The init expressions are evaluated (in some unspecified order), the variables are bound to fresh locations, the results of the init expressions are stored in the bindings of the variables, and then the iteration phase begins.

Each iteration begins by evaluating test; if the result is false (see section 6.3), then the command expressions are evaluated in order for effect, the step expressions are evaluated in some unspecified order, the variables are bound to fresh locations, the results of the steps are stored in the bindings of the variables, and the next iteration begins.

If test evaluates to a true value, then the expressions are evaluated from left to right and the values of the last expression are returned. If no expressions are present, then the value of the `=13`do expression is unspecified.

The regionindexfile"region" "rm" aux of the binding of a variable consists of the entire `=13`do expression except for the inits. It is an error for a variable to appear more than once in the list of `=13`do variables.

A step can be omitted, in which case the effect is the same as if `=13`(variable init variable) had been written instead of `=13`(variable init).

`=13`=0.5em(letvariable bindings body) syntax

Semantics: ``Named `=13`let'' is a variant on the syntax of indexfile"let" "tt" aux let which provides a more general looping construct than `=13`do and can also be used to express recursion. It has the same syntax and semantics as ordinary `=13`let except that variable is bound within body to a procedure whose formal arguments are the bound variables and whose body is body. Thus the execution of body can be repeated by invoking the procedure named by variable.
