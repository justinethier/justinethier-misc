Output

If port is omitted from any output procedure, it defaults to the value returned by `=13`(current-output-port). It is an error to attempt an output operation on a closed port.






`=13`=0.5em(indexfile"write" "tt" main writeobj) write library procedure
`=13`=0.5em(writeobj port) write library procedure

Writes a representation of obj to the given textual output port. Strings that appear in the written representation are enclosed in quotation marks, and within those strings backslash and quotation mark characters are escaped by backslashes. Symbols that contain non-ASCII characters are escaped with vertical lines. Character objects are written using the `=13`#\ notation.

If obj contains cycles which would cause an infinite loop using the normal written representation, then at least the objects that form part of the cycle must be represented using datum labels as described in section 2.4. Datum labels must not be used if there are no cycles.

Implementations may support extended syntax to represent record types or other types that do not have datum representations.

The `=13`write procedure returns an unspecified value.






`=13`=0.5em(indexfile"write-shared" "tt" main write-sharedobj) write library procedure
`=13`=0.5em(write-sharedobj port) write library procedure

The `=13`write-shared procedure is the same as `=13`write, except that shared structure must be represented using datum labels for all pairs and vectors that appear more than once in the output.






`=13`=0.5em(indexfile"write-simple" "tt" main write-simpleobj) write library procedure
`=13`=0.5em(write-simpleobj port) write library procedure

The `=13`write-simple procedure is the same as `=13`write, except that shared structure is never represented using datum labels. This can cause `=13`write-simple not to terminate if obj contains circular structure.






`=13`=0.5em(indexfile"display" "tt" main displayobj) write library procedure
`=13`=0.5em(displayobj port) write library procedure

Writes a representation of obj to the given textual output port. Strings that appear in the written representation are output as if by `=13`write-string instead of by `=13`write. Symbols are not escaped. Character objects appear in the representation as if written by `=13`write-char instead of by `=13`write.

The `=13`display representation of other objects is unspecified. However, `=13`display must not loop forever on self-referencing pairs, vectors, or records. Thus if the normal `=13`write representation is used, datum labels are needed to represent cycles as in `=13`write.

Implementations may support extended syntax to represent record types or other types that do not have datum representations.

The `=13`display procedure returns an unspecified value.

Rationale: The `=13`write procedure is intended for producing machine-readable output and `=13`display for producing human-readable output.





`=13`=0.5em(indexfile"newline" "tt" main newline) procedure
`=13`=0.5em(newlineport) procedure

Writes an end of line to textual output port. Exactly how this is done differs from one operating system to another. Returns an unspecified value.






`=13`=0.5em(indexfile"write-char" "tt" main write-charchar) procedure
`=13`=0.5em(write-charchar port) procedure

Writes the character char (not an external representation of the character) to the given textual output port and returns an unspecified value.






`=13`=0.5em(indexfile"write-string" "tt" main write-stringstring) procedure
`=13`=0.5em(write-stringstring port) procedure
`=13`=0.5em(write-stringstring port start) procedure
`=13`=0.5em(write-stringstring port start end) procedure

Writes the characters of string from start to end in left-to-right order to the textual output port.






`=13`=0.5em(indexfile"write-u8" "tt" main write-u8byte) procedure
`=13`=0.5em(write-u8byte port) procedure

Writes the byte to the given binary output port and returns an unspecified value.






`=13`=0.5em(indexfile"write-bytevector" "tt" main write-bytevectorbytevector) procedure
`=13`=0.5em(write-bytevectorbytevector port) procedure
`=13`=0.5em(write-bytevectorbytevector port start) procedure
`=13`=0.5em(write-bytevectorbytevector port start end) procedure

`=13`=0.5em(indexfile"flush-output-port" "tt" main flush-output-port) procedure
`=13`=0.5em(flush-output-portport) procedure

Flushes any buffered output from the buffer of output-port to the underlying file or device and returns an unspecified value.
