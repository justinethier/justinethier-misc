Equivalence predicates

A indexfile"predicate" "rm" main predicate is a procedure that always returns a boolean value (#t or #f). An indexfile"equivalence predicate" "rm" main equivalence predicate is the computational analogue of a mathematical equivalence relation; it is symmetric, reflexive, and transitive. Of the equivalence predicates described in this section, `=13`eq? is the finest or most discriminating, `=13`equal? is the coarsest, and `=13`eqv? is slightly less discriminating than `=13`eq?.





`=13`=0.5em(indexfile"eqv?" "tt" main eqv?obj obj) procedure

The `=13`eqv? procedure defines a useful equivalence relation on objects. Briefly, it returns #t if obj and obj are normally regarded as the same object. This relation is left slightly open to interpretation, but the following partial specification of `=13`eqv? holds for all implementations of Scheme.

The `=13`eqv? procedure returns #t if:

  • obj and obj are both #t or both #f.

  • obj and obj are both symbols and are the same symbol according to the `=13`symbol=? procedure (section 6.5).

  • obj and obj are both exact numbers and are numerically equal (in the sense of `=13`=).

  • obj and obj are both inexact numbers such that they are numerically equal (in the sense of `=13`=) and they yield the same results (in the sense of `=13`eqv?) when passed as arguments to any other procedure that can be defined as a finite composition of Scheme's standard arithmetic procedures, provided it does not result in a NaN value.

  • obj and obj are both characters and are the same character according to the `=13`char=? procedure (section 6.6).

  • obj and obj are both the empty list.

  • obj and obj are pairs, vectors, bytevectors, records, or strings that denote the same location in the store (section 3.4).

  • obj and obj are procedures whose location tags are equal (section 4.1.4).

The `=13`eqv? procedure returns #f if:

  • obj and obj are of different types (section 3.2).

  • one of obj and obj is #t but the other is #f.

  • obj and obj are symbols but are not the same symbol according to the `=13`symbol=? procedure (section 6.5).

  • one of obj and obj is an exact number but the other is an inexact number.

  • obj and obj are both exact numbers and are numerically unequal (in the sense of `=13`=).

  • obj and obj are both inexact numbers such that either they are numerically unequal (in the sense of `=13`=), or they do not yield the same results (in the sense of `=13`eqv?) when passed as arguments to any other procedure that can be defined as a finite composition of Scheme's standard arithmetic procedures, provided it does not result in a NaN value. As an exception, the behavior of `=13`eqv? is unspecified when both obj and obj are NaN.

  • obj and obj are characters for which the `=13`char=? procedure returns #f.

  • one of obj and obj is the empty list but the other is not.

  • obj and obj are pairs, vectors, bytevectors, records, or strings that denote distinct locations.

  • obj and obj are procedures that would behave differently (return different values or have different side effects) for some arguments.

The `=13`gen-counter procedure must return a distinct procedure every time, since each procedure has its own internal counter. The `=13`gen-loser procedure, however, returns operationally equivalent procedures each time, since the local state does not affect the value or side effects of the procedures. However, `=13`eqv? may or may not detect this equivalence.

Since it is an error to modify constant objects (those returned by literal expressions), implementations may share structure between constants where appropriate. Thus the value of `=13`eqv? on constants is sometimes implementation-dependent.


unspecified error makeother^`=13` 


          gobblecr(eqv? '(a) '(a))                 (eqv? "a" "a")                   (eqv? '(b) (cdr '(a b)))  (let ((x '(a)))  (eqv? x x))                    #t

The above definition of `=13`eqv? allows implementations latitude in their treatment of procedures and literals: implementations may either detect or fail to detect that two procedures or two literals are equivalent to each other, and can decide whether or not to merge representations of equivalent objects by using the same pointer or bit pattern to represent both.

Note: If inexact numbers are represented as IEEE binary floating-point numbers, then an implementation of `=13`eqv? that simply compares equal-sized inexact numbers for bitwise equality is correct by the above definition.






`=13`=0.5em(indexfile"eq?" "tt" main eq?obj obj) procedure

The `=13`eq? procedure is similar to `=13`eqv? except that in some cases it is capable of discerning distinctions finer than those detectable by `=13`eqv?. It must always return #f when `=13`eqv? also would, but may return #f in some cases where `=13`eqv? would return #t.

On symbols, booleans, the empty list, pairs, and records, and also on non-empty strings, vectors, and bytevectors, `=13`eq? and `=13`eqv? are guaranteed to have the same behavior. On procedures, `=13`eq? must return true if the arguments' location tags are equal. On numbers and characters, `=13`eq?'s behavior is implementation-dependent, but it will always return either true or false. On empty strings, empty vectors, and empty bytevectors, `=13`eq? may also behave differently from `=13`eqv?.

Rationale: It will usually be possible to implement `=13`eq? much more efficiently than `=13`eqv?, for example, as a simple pointer comparison instead of as some more complicated operation. One reason is that it is not always possible to compute `=13`eqv? of two numbers in constant time, whereas `=13`eq? implemented as pointer comparison will always finish in constant time.






`=13`=0.5em(indexfile"equal?" "tt" main equal?obj obj) procedure

The `=13`equal? procedure, when applied to pairs, vectors, strings and bytevectors, recursively compares them, returning #t when the unfoldings of its arguments into (possibly infinite) trees are equal (in the sense of `=13`equal?) as ordered trees, and #f otherwise. It returns the same as `=13`eqv? when applied to booleans, symbols, numbers, characters, ports, procedures, and the empty list. If two objects are `=13`eqv?, they must be `=13`equal? as well. In all other cases, `=13`equal? may return either #t or #f.

Even if its arguments are circular data structures, `=13`equal? must always terminate.
