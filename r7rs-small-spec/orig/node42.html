Syntax: Each cond clause is as in the specification of `=13`cond.

Semantics: The body is evaluated with an exception handler that binds the raised object (see indexfile"raise" "tt" aux raise in section 6.11) to variable and, within the scope of that binding, evaluates the clauses as if they were the clauses of a `=13`cond expression. That implicit `=13`cond expression is evaluated with the continuation and dynamic environment of the `=13`guard expression. If every cond clause's test evaluates to #f and there is no else clause, then `=13`raise-continuable is invoked on the raised object within the dynamic environment of the original call to `=13`raise or `=13`raise-continuable, except that the current exception handler is that of the `=13`guard expression.

See section 6.11 for a more complete discussion of exceptions.
