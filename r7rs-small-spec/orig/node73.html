Pairs and lists

A indexfile"pair" "rm" main pair (sometimes called a indexfile"dotted pair" "rm" main dotted pair) is a record structure with two fields called the car and cdr fields (for historical reasons). Pairs are created by the procedure `=13`cons. The car and cdr fields are accessed by the procedures `=13`car and `=13`cdr. The car and cdr fields are assigned by the procedures `=13`set-car! and `=13`set-cdr!.

Pairs are used primarily to represent lists. A indexfile"list" "rm" main list can be defined recursively as either the empty listindexfile"empty list" "rm" aux or a pair whose cdr is a list. More precisely, the set of lists is defined as the smallest set X such that

  • The empty list is in X.
  • If list is in X, then any pair whose cdr field contains list is also in X.

The objects in the car fields of successive pairs of a list are the elements of the list. For example, a two-element list is a pair whose car is the first element and whose cdr is a pair whose car is the second element and whose cdr is the empty list. The length of a list is the number of elements, which is the same as the number of pairs.

The empty listindexfile"empty list" "rm" main is a special object of its own type. It is not a pair, it has no elements, and its length is zero.

Note: The above definitions imply that all lists have finite length and are terminated by the empty list.

The most general notation (external representation) for Scheme pairs is the ``dotted'' notation `=13`(c . c) where c is the value of the car field and c is the value of the cdr field. For example `=13`(4 . 5) is a pair whose car is 4 and whose cdr is 5. Note that `=13`(4 . 5) is the external representation of a pair, not an expression that evaluates to a pair.

A more streamlined notation can be used for lists: the elements of the list are simply enclosed in parentheses and separated by spaces. The empty listindexfile"empty list" "rm" aux is written (). For example,


unspecified error makeother^`=13` 


          gobblecr(a b c d e)

and


unspecified error makeother^`=13` 


          gobblecr(a . (b . (c . (d . (e . ())))))

are equivalent notations for a list of symbols.

A chain of pairs not ending in the empty list is called an indexfile"improper list" "rm" main improper list. Note that an improper list is not a list. The list and dotted notations can be combined to represent improper lists:


unspecified error makeother^`=13` 


          gobblecr(a b c . d)

is equivalent to


unspecified error makeother^`=13` 


          gobblecr(a . (b . (c . d)))

Whether a given pair is a list depends upon what is stored in the cdr field. When the indexfile"set-cdr!" "tt" aux set-cdr! procedure is used, an object can be a list one moment and not the next:


unspecified error makeother^`=13` 


          gobblecr(define x (list 'a 'b 'c))(define y x)y                       (a b c)(list? y)               #t(set-cdr! x 4)          (a . 4)(eqv? x y)              #ty                       (a . 4)(list? y)               #f(set-cdr! x x)          (list? x)               #f

Within literal expressions and representations of objects read by the indexfile"read" "tt" aux read procedure, the forms 'datumindexfile"'" "tt" aux , datum, ,datumindexfile"," "tt" aux , and ,@datum denote two-element lists whose first elements are the symbols indexfile"quote" "tt" aux quote, indexfile"quasiquote" "tt" aux quasiquote, indexfile"unquote" "tt" aux unquote, and indexfile"unquote-splicing" "tt" aux unquote-splicing, respectively. The second element in each case is datum. This convention is supported so that arbitrary Scheme programs can be represented as lists. That is, according to Scheme's grammar, every expression is also a datum (see section 7.1.2). Among other things, this permits the use of the `=13`read procedure to parse Scheme programs. See section 3.3.

`=13`=0.5em(indexfile"append" "tt" main appendlist ) procedure

The last argument, if there is one, can be of any type.

Returns a list consisting of the elements of the first list followed by the elements of the other lists. If there are no arguments, the empty list is returned. If there is exactly one argument, it is returned. Otherwise the resulting list is always newly allocated, except that it shares structure with the last argument. `=13`=0.5em(indexfile"assq" "tt" main assqobj alist) procedure
`=13`=0.5em(indexfile"assv" "tt" main assvobj alist) procedure
`=13`=0.5em(indexfile"assoc" "tt" main assocobj alist) procedure
`=13`=0.5em(assocobj alist compare) procedure

It is an error if alist (for ``association list'') is not a list of pairs.

These procedures find the first pair in alist whose car field is obj, and returns that pair. If no pair in alist has obj as its car, then #f (not the empty list) is returned. The `=13`assq procedure uses `=13`eq? to compare obj with the car fields of the pairs in alist, while `=13`assv uses `=13`eqv? and `=13`assoc uses compare if given and `=13`equal? otherwise.


unspecified error makeother^`=13` 


          gobblecr(define e '((a 1) (b 2) (c 3)))(assq 'a e)     (a 1)(assq 'b e)     (b 2)(assq 'd e)     #f(assq (list 'a) '(((a)) ((b)) ((c))))                #f(assoc (list 'a) '(((a)) ((b)) ((c))))                            ((a))(assoc 2.0 '((1 1) (2 4) (3 9)) =)                           (2 4)(assq 5 '((2 3) (5 7) (11 13)))                            (assv 5 '((2 3) (5 7) (11 13)))                            (5 7)

Rationale: Although they are often used as predicates, `=13`memq, `=13`memv, `=13`member, `=13`assq, `=13`assv, and `=13`assoc do not have question marks in their names because they return potentially useful values rather than just #t or #f.





`=13`=0.5em(indexfile"list-copy" "tt" main list-copyobj) procedure

Returns a newly allocated copy of the given obj if it is a list. Only the pairs themselves are copied; the cars of the result are the same (in the sense of `=13`eqv?) as the cars of list. If obj is an improper list, so is the result, and the final cdrs are the same in the sense of `=13`eqv?. An obj which is not a list is returned unchanged. It is an error if obj is a circular list.


unspecified error makeother^`=13` 


          gobblecr(define a '(1 8 2 8)) ; a may be immutable(define b (list-copy a))(set-car! b 3)        ; b is mutableb (3 8 2 8)a (1 8 2 8)
