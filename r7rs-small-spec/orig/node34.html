Inclusion





`=13`=0.5em(indexfile"include" "tt" main includestring string ) syntax
`=13`=0.5em(include-cistring string ) syntax

Semantics: Both indexfile"include" "tt" aux include and indexfile"include-ci" "tt" aux include-ci take one or more filenames expressed as string literals, apply an implementation-specific algorithm to find corresponding files, read the contents of the files in the specified order as if by repeated applications of `=13`read, and effectively replace the `=13`include or `=13`include-ci expression with a `=13`begin expression containing what was read from the files. The difference between the two is that indexfile"include-ci" "tt" aux include-ci reads each file as if it began with the `=13`#!fold-case directive, while indexfile"include" "tt" aux include does not.

Note: Implementations are encouraged to search for files in the directory which contains the including file, and to provide a way for users to specify other directories to search.
